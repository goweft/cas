### Block 3: Sub-agent decomposition

**internal/agent package** (`682c886`)
Three named agents, each owning one category of LLM call with a Design by
Contract enforcement layer.

`GenerationAgent` (KindCreate)
- Pre: wsType ∈ {document,code,list}, prompt non-empty, title non-empty
- Post: content non-empty, content ≤ 512 KB
- `Generate()` and `Stream()`

`EditAgent` (KindEdit)
- Pre: wsType allowed, current content non-empty, edit request non-empty
- Post: result non-empty, result ≤ 512 KB, result ≥ 10% of original (truncation guard)
- `Edit()` and `Stream()`

`CombineAgent` (KindCombine)
- Pre: ≥ 2 sources, all sources non-empty
- Post: result non-empty, result ≤ 512 KB
- `Combine()` and `Stream()`

Shell updated: `handleCreate/streamCreate` → `genAgent`, `handleEdit/streamEdit`
→ `editAgent`, `handleCombine/streamCombine` → `combAgent`. All direct
`llm.Complete/Stream` calls for workspace ops removed from shell.
`handleChat`/`streamChat` still call llm directly (chat has no workspace op).

Tests: 8 contract violation tests (bad wsType, empty inputs, too few sources).
